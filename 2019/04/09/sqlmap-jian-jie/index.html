SQLMap参数说明

-v
输出级别

0:只显示错误信息和关键信息

1:同时显示普通信息和警告信息(默认)

2:同时显示debug信息

3:同时显示注入使用的攻击载荷

4:同时显示HTTP请求

5:同时显示HTTP请求头

6:同时显示HTTP相应体

-d
直接连接数据库

-u
指定目标URL

Demo:

    python sqlmap.py -u "http://192.168.1.1:8080/user.php?id=0"

-method

指定HTTP请求方法

Demo:

    -method=post

-data

该参数指定的数据会被作为POST数据提交,sqlmap也会检测该参数指定数据是否存在sql注入漏洞

Demo:


    python sqlmap.py -u 'http://192.168.1.1:8080/user.php' --data="id=0&name=user"

-para-del

指定POST请求数据的分隔符

默认的分隔符是"&",可以使用-param-del指定其他的分隔符

-random-agent

设置随机请求头

-Host

手动指定Http头的HOST值

-Referer

手动指定HTTP头重的Referer的值

-cookie

某些网站需要cookie才能够访问,这个参数可以用来指定Cookie值

Demo:

python sqlmap.py -u "http://192.168.1.1:8080/user.php" --cookie "浏览器中的cookie值"

-drop-set-cookie

若浏览器响应头中有"set-cookie",sqlmap会自动设置"set-cookie",设置的cookie,并对这些cookie进行检测,若不想让sqlmap这样做,添加参数"drop-set-cookie"即可,这样sqlmap就会忽略"set-cookie"

-o
优化SQLMap的性能
此参数默认会优化三个参数
-keep-alive(HTTP长链接)
-null-connection(HTTP空链接)
-threads=3(HTTP并发)

-p
注入相关的参数,用于指定要测试的参数,定制攻击载荷和选择篡改脚本

Demo

    只测试id和user-agent
    -p "id,user-agent"

-skip

不想测试某些参数

Demo
    不想测试user-agent,referer

    --skip="user-agent,referer"

-dbms

指定数据库管理系统

搭配fingerprint可以对数据库进行指纹识别

Demo
    -dbms 数据库名

Mysql和MSSQLServer指定的方法
    --dbms Mysql

-os

指定运行数据库的操作系统,默认情况下SQLMap会自动扫描运行数据库的操作系统,如果不是非常确定的情况下,还是让SQLMap自己扫描会更好些

– level
Polyload等级
